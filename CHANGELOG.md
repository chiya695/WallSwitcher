# 壁纸切换器更新日志

## v1.3.0 (2026-05-13)

这次主要是适配 Android 16 (API 36)，顺便修了一些攒了很久的小问题。

### Android 16 适配

- compileSdk 和 targetSdk 升到 36
- 加了 `enableEdgeToEdge()`，Android 16 不让退出全屏模式了
- 给壁纸切换服务声明了 `foregroundServiceType="specialUse"`，加上了对应的权限
- Navigation Compose 升到 2.9.0，支持预测性返回手势

### 依赖升级

- Compose BOM: 2024.09.00 → 2025.05.00
- Navigation Compose: 2.7.7 → 2.9.0
- WorkManager: 2.9.0 → 2.10.0
- Coil: 2.5.0 → 2.7.0
- Accompanist Permissions: 0.32.0 → 0.37.0
- AndroidX Core KTX: 1.15.0 → 1.16.0
- Lifecycle Runtime KTX: 2.8.7 → 2.9.0

### Bug 修复

- CropMethod 枚举在两个文件里各定义了一份，删掉了 Settings.kt 里重复的
- GalleryScreen 里 LogUtils.log() 在 Composable 重组时被反复调用，日志文件会爆，删掉了
- WorkManager 同时手动初始化又实现了 Configuration.Provider，冲突了，删掉手动初始化
- WallpaperUtils 裁剪后原始 Bitmap 没 recycle，内存泄漏，加上了
- MainViewModel 里 checkServiceRunning() 之前硬编码返回 false，改成了 ActivityManager 查询
- SettingsScreen 和 GalleryScreen 里的硬编码中文字符串换成了 stringResource

## v1.2.1 (2025-04-04)

### 新增功能

- **图片质量设置**：添加了图片质量设置选项，允许用户选择是否启用图片压缩以及压缩质量
  - 高质量模式：使用2560x1440分辨率和ARGB_8888格式，保留更多细节
  - 平衡模式（默认）：使用1920x1080分辨率和RGB_565格式，平衡质量和性能
  - 省内存模式：使用1280x720分辨率和RGB_565格式，最大限度节省内存
  - 禁用压缩：使用原始图片质量，不进行任何压缩或降采样

### 性能优化

- **分页加载机制**：添加了分页加载壁纸的功能，大幅提高了处理大量图片时的性能
- **懒加载图片**：优化了图库界面，实现了图片的懒加载，减少内存占用
- **图片预加载**：添加了图片预加载机制，提高浏览体验
- **内存缓存**：实现了图片缓存系统，避免重复加载同一图片

### 图片处理优化

- **采样率优化**：根据设备屏幕尺寸动态计算图片采样率，减少内存占用
- **位图配置优化**：使用RGB_565配置代替ARGB_8888，每像素节省50%内存
- **最大尺寸限制**：限制加载图片的最大尺寸为1920x1080像素，平衡质量和性能

### 小部件优化

- **后台处理**：使用WorkManager替代直接在BroadcastReceiver中处理图片，提高稳定性
- **进度通知**：添加了壁纸切换进度通知，提升用户体验
- **错误处理**：改进了错误处理机制，提高应用稳定性

### 图片质量说明

当前版本在图片处理时采用了以下优化策略：

1. **采样率缩减**：根据设备屏幕和原图尺寸计算合适的采样率，减少内存中的像素数量
2. **位图配置优化**：使用RGB_565配置，比默认的ARGB_8888每像素少用一半内存，但会牺牲一些色彩精度
3. **最大尺寸限制**：限制加载的最大尺寸为1920x1080像素，对于大多数手机屏幕已经足够

**注意事项**
当用户关闭"启用图片压缩"选项后，虽然可以获得最高质量的壁纸效果，但也有一些需要注意的地方：
- 内存占用增加：原始图片通常分辨率很高，可能会占用大量内存，特别是在处理多张图片时
- 处理速度变慢：加载和处理高分辨率图片需要更多时间，可能导致壁纸切换速度变慢
- 可能出现OOM错误：如果图片太大，可能会导致内存溢出(Out Of Memory)错误
